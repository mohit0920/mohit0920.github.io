Constraints

used to restrict the behaviour of columns.

These rules defines values allowed in columns.

Also limits the type of data that can go after a column.

Constraints can be created when one define a table using sql create statement.

Or after the table is created with sql alter statement.

Through constraints user can implement business rules to the table through which user can ensure Data integrity and Data consistency.

By Ensuring what type of data goes into the table constarints can ensure the accuracy and reliablity of the data.

Can be Defined as inline or out of line constraints.

  1. Inline Or column level - defined in same line as of its column. Can only be defined at the time of table creation 

    xxxxxxxxxx
    create table table_name (
      col1 data_type(size) constraint_type,
      col2 data_type(size) constraint_type
    );

  2. Out of line - Defined in its own line. These constraints can be defined at the time of table creation as well as after table creation.

    xxxxxxxxxx
    CONSTRAINT constraint_name constraint_type reference_col

    Ex

    xxxxxxxxxx
    CONSTRAINT not_null NOT NULL (col1, col2);

    We can Enable, Disable, modify, drop or rename a constraint using alter command. 

    xxxxxxxxxx
    alter table drop CONSTRAINT constraint_name;

Impact of adding new constraint and modifying a constraint

When we apply not null constraint on a column where already it has few null values then a error pops up with violation of data integrity.

Solution

Truncate table apply alter or constraint load data.

Replace null values with appropriate values.

Types of Constraints

List of Constraints Supported by Oracle database.

  1. Check Costraint -

    limits the value range that can be placed in a column. Enforce domain integrity all values in the column must statisfies some certain constraints. Can apply on single column or on table to allow some set of values to entered into the Column based on the other column values like if there is three columns min max and avg then avg must be in between min and max.

    Can not refer to any column in other tables. Can not include sub query.

    xxxxxxxxxx
    create table student (
      St_id int,
      name varchar2(30),
      age int check (age >= 10 and age <= 20)
    );

  2. Unique -

    Ensure a column values must be unique for every row. Allow null values in column and even null values can be inserted multiple times. differ from primary key in allowing null values.

  3. Not null

  4. Primary key -

    used to uniquely identify each row in database. doesn't allow null values. Can be either simple or composite key. It cannot have more than 32 columns as composite primary key, cannot be defined columns having LOB, LONG , LONG RAW, BFILE .. etc. datatypes.

    Example for composite primary key.

    xxxxxxxxxx
    create table student (
      st_id int,
      course_id int,
      name char(5),
      PRIMARY KEY (st_id, course_id)
    )

  5. Foreign key or Referntial key -

    Foreign key in one table points to primary key in another table. Foreign key except null cannot be inserted into the table with no candidate key in the linked table. Primary key and foreign key can exist in same table. Data type must be same in primary table and linked table. Composite primary key must refer to composite primary key in primary or parent table. Means any column which is declared as Foreign key can contain

    1. Null
    2. Primary key from linked table
    3. Candidate key

    Deleting a record in the primary table which have linked record in the linked table will break referntial integrity.

    Cascading update and cascading delete can be used to ensure that changes made in the linked table are refelected in the primary table.

    xxxxxxxxxx
    CREATE TABLE table2 (col4 DATATYPE(size), col5 DATATYPE(size), col6 DATATYPE(size), ........, CONSTRAINT cons_name FOREIGN KEY(col4) REFERENCES table1(col1));

    Composite foreign key

    xxxxxxxxxx
    CREATE TABLE table2 (col4 DATATYPE(size), col5 DATATYPE(size), col6 DATATYPE(size), ........, CONSTRAINT cons_name FOREIGN KEY (col4,col5) REFERENCES table1(col1,col2));

    while creating child or linked table we can define on delete cascade mean what to do if try to delete a row from parent table where which we used here foreign key.

    ON DELETE SET NULL
    xxxxxxxxxx
    CREATE TABLE table_name (colname1 DATATYPE(size), colname2 DATATYPE(size),
    CONSTRAINT cons_name FOREIGN KEY (colname1,colname2) REFERENCES table1(col1,col2) ON DELETE SET NULL);

    On delete cascade

    Means if row from parent table is deleted delete the corresponding rows from child table also.

    xxxxxxxxxx
    CREATE TABLE person(name VARCHAR2(30),dob DATE,gender
    CHAR(1),address NUMBER(4) REFERENCES location(loc_id) ON DELETE
    CASCADE, gender CHAR(1));

  6. Table and column level constraints.

    Table level constraints that aplied to complete table. If there is any modification to the row of the table regardless of the column check constraint. Check and Primary key are example of table level.

    Column level constraint is checked whenever the column value is changed or inserted.

    Sum up table and column level

    If a contraint depends on other columns or we need to check other columns and rows than that is table level.
